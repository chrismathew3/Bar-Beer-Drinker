Bar Beer Drinker Basic Run Down: Successes & Failures

  1. Bar Page
    • Initially clicking on the bars tab on the nav bar will bring you to a page which shows all bars in the database.
    • All the bars were created using a python script and some python modules used to generate fake but realistic data.
    • Clicking on a bar's name will bring you to that specific bar's component page.
    • Initially displayed will be that bar's specific details.
    • Afterwards three charts will be displayed showing the top drinkers/spender, most popular beers, and the manufacturers who sell the most beer all for that specific bar.
  2. Beer Page
    • Initially clicking on the beer tab on the nav bar will bring you to a page which shows all beers in the database.
    • All the beers are real along with their manufacturers.
    • Clicking on a beer's name will bring you to that specific beer's component page.
    • Initially displayed will be that beer's specific details.
    • Afterwards two charts will be displayed showing where the beer sells the most, and the biggest consumers of the beer.
  3. Drinker Page
    • Initially clicking on the drinkers tab on the nav bar will bring you to a page which shows all drinkers in the database.
    • All the drinkers were created using a python script and some python modules used to generate fake but realistic data similar to how the bars were generated.
    • Clicking on a drinker's name will bring you to that specific drinker's component page.
    • Initially displayed will be that drinker's specific details.
    • Afterwards one table and two charts will be displayed showing all the transactions the drinker made, the distribution of beer bought, and the distribution of spending at various bars.
  4. Problems/Failures
    • I did not incorporate a date feature in the database so I was not able to provide a time distribution.
    • I did not build out a SQL Interface due to lack of time.
    • I generate CSV files for all entities and entity relationships, and all were uploaded to the database except frequents because for some reason I was getting an "List index out of range" error.
      Had I know this would of happened earlier on, I would of found a way to fix this bug, but I had already generated the data by then and if I altered the frequents table then the assertions on the data would be inaccurate
  5. Long-Term Plan
